---
name: project-planning
description: "Guidelines for planning and designing software projects"
---

When planning a software project, follow these steps:

1. Clarify requirements: Identify what the user actually needs vs what they asked for. Ask questions if the scope is ambiguous.
2. Break down the work: Split the project into discrete, independently testable components.
3. Identify dependencies: Determine which components depend on others and plan the build order accordingly.
4. Choose the right tools: Select technologies based on project requirements, not personal preference. Justify each choice.
5. Define interfaces first: Agree on API contracts and data shapes before implementing internals.
6. Plan for testing: Every component should have a clear verification strategy.
7. Consider edge cases: Think about error handling, empty states, and failure modes upfront.
